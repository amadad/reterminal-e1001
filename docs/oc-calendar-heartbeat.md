# Calendar heartbeat: keep `calendar.md` fresh

The reTerminal `CalendarProvider` reads `~/reterminal-content/family/calendar.md`
and renders slot 0. The file can be written by any local scheduler or calendar
exporter. The display pipeline itself does not talk to Google Calendar, iCloud,
or any other calendar API.

## Example scheduled task

Add a task to your scheduler of choice:

```text
## refresh-kitchen-calendar (every 20 min)

Pull the household calendar and write ~/reterminal-content/family/calendar.md.

Steps:
1. Fetch events from your calendar source.
2. Render the result into the file format below.
3. Atomic write: write to calendar.md.tmp, then rename to calendar.md.

Failure handling: if the calendar fetch errors, do not touch calendar.md. Log
the error and let the display keep showing the last-good calendar.
```

## `calendar.md` format

Renderer reads `## Today` and `## Tomorrow` only. `## Notes` is ignored and is a
good place to mark the file as machine-generated.

```markdown
# Today

Today + tomorrow agenda for the kitchen display (slot 0). Renderer reads the
## Today and ## Tomorrow sections. Auto-generated from a calendar exporter — do
not edit by hand if your exporter overwrites this file.

Format per line: `- HH:MM[am|pm] Label [@who]`. The `[@who]` tag is optional
and lowercase (for example `[@kid1]`, `[@family]`). Skip all-day events whose
label adds nothing to the wall display.

## Today

- 9:30am Piano [@kid1]
- 12:00pm Family lunch
- 4:00pm Baseball practice [@kid2]

## Tomorrow

- 8:00am School
- 6:30pm Movie night

## Notes

Auto-generated by a local calendar exporter. Last refresh: 2026-04-27 09:30.
```

## Cadence

20–30 minutes is a good default. Faster wastes calls; slower means late-added
appointments may appear too late. The reTerminal pipeline reacts within about a
second of a file write when running with `publish --watch`, so end-to-end
latency is bounded by the exporter cadence.

## Why this is outside reterminal

Failure isolation. If calendar auth expires or a provider has an outage:

- The breakage stays in the exporter/scheduler.
- `calendar.md` does not get corrupted.
- The reTerminal pipeline keeps running for the other slots.
- The display shows the last-good calendar instead of an error state.
