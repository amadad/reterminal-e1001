# OC heartbeat: keep `calendar.md` fresh

The reTerminal CalendarProvider reads `~/madad/family/calendar.md` and
renders slot 0. The file is machine-written by an OpenClaw heartbeat task
that calls the `gws` skill — the reterminal pipeline never talks to Google
directly. This doc is the OC-side recipe that keeps that file fresh.

## Heartbeat task to add

Add to `~/oc-min/HEARTBEAT.md`:

```
## refresh-kitchen-calendar (every 20 min)

Pull the family Google Calendar and write ~/madad/family/calendar.md.

Steps:
1. gws calendar list --from today --to "tomorrow 23:59" \
     --calendar family13940637679946115055@group.calendar.google.com
2. Render the result into the file format below (see "calendar.md format").
3. Atomic write: write to ~/madad/family/calendar.md.tmp, then rename.

Failure handling: if gws errors (auth expired, network), do NOT touch
calendar.md. Log the error to OC, continue. The kitchen display will
keep showing the last-good calendar; humans can re-auth gws on their
own time. Do not retry — the next heartbeat tick will try again.
```

## `calendar.md` format

Match the conventions used by `missions.md` / `events.md` / `activities.md`
(see `~/madad/family/CONVENTIONS.md`). Renderer reads `## Today` and
`## Tomorrow` only; `## Notes` is ignored and is where you mark the file as
machine-generated.

```markdown
# Today

Today + tomorrow agenda for the kitchen display (slot 0). Renderer reads
the ## Today and ## Tomorrow sections. Auto-generated from Google Calendar
by the OC heartbeat — do not edit by hand, edits are overwritten on the
next tick.

Format per line: `- HH:MM[am|pm] Label [@who]`. The `[@who]` tag is
optional and lowercase (e.g. `[@hasan]`, `[@laila]`). Skip all-day
events whose label adds nothing to the wall display.

## Today

- 9:30am Hasan piano [@hasan]
- 12:00pm Family lunch
- 4:00pm Ammar baseball practice [@ammar]

## Tomorrow

- 8:00am School
- 6:30pm Movie night

## Notes

Auto-generated by OC heartbeat from gws calendar. Do not edit by hand —
edits will be overwritten on the next tick. Last refresh: 2026-04-27 09:30.
```

## Cadence

20–30 minutes is the sweet spot. Faster wastes calls; slower means a
late-added doctor's appointment shows up after lunch. The reterminal
pipeline reacts within ~1 second of file write (FSEvents), so the
end-to-end latency is bounded by the heartbeat tick.

## Why this is in OC and not in reterminal

Failure isolation. If gws auth expires or Google has an outage:

- The breakage stays in OC's lane. OC notices and reports.
- `calendar.md` does not get corrupted (atomic write or no write at all).
- The reterminal pipeline keeps running for the other three slots —
  missions/events/activities are unaffected.
- The kitchen display shows the last-good calendar instead of an error
  state.

If we put the gws call inside CalendarProvider, an auth glitch would
propagate into the display loop, retries would compound, and the failure
mode would be tangled. Keeping it as a one-way write to a file is the
clean separation.
